# BE-005: Supabase Auth設定

## 基本情報

- **フェーズ**: 3 - 認証システム
- **優先度**: Critical
- **見積もり**: 3h
- **前提条件**: BE-001完了
- **タイプ**: Backend

## 実装内容

### 主要タスク

- [ ] マジックリンク認証有効化
- [ ] メールテンプレートカスタマイズ（日本語対応）
- [ ] リダイレクトURL設定（rovrov://auth/callback）
- [ ] JWT有効期限設定（access: 1h, refresh: 30d）
- [ ] auth.users作成時のprofilesレコード自動作成トリガー
- [ ] セッション管理設定

### 詳細設定

1. **マジックリンク認証**:
   - Supabaseダッシュボードでマジックリンク認証を有効化
   - カスタムSMTP設定（日本語メール対応）
   - リンク有効期限設定（デフォルト1時間）

2. **メールテンプレート**:
   - 日本語件名・本文設定
   - RovRovブランディング適用
   - モバイルアプリ用リダイレクトURL組み込み

3. **JWT設定**:
   - Access Token: 1時間有効
   - Refresh Token: 30日有効
   - 自動更新設定

4. **プロファイル自動作成**:

   ```sql
   CREATE OR REPLACE FUNCTION handle_new_user()
   RETURNS trigger AS $$
   BEGIN
     INSERT INTO public.profiles (id, username, display_name, created_at)
     VALUES (NEW.id, COALESCE(NEW.raw_user_meta_data->>'username', 'user_' || substr(NEW.id::text, 1, 8)),
             COALESCE(NEW.raw_user_meta_data->>'display_name', 'New User'), NOW());
     RETURN NEW;
   END;
   $$ LANGUAGE plpgsql SECURITY DEFINER;

   CREATE TRIGGER on_auth_user_created
     AFTER INSERT ON auth.users
     FOR EACH ROW EXECUTE PROCEDURE handle_new_user();
   ```

   CREATE INDEX idx_profiles_created_at ON public.profiles(created_at DESC);

2. **自動プロファイル作成トリガー実装**

   ```sql
   -- handle_new_user関数
   CREATE OR REPLACE FUNCTION public.handle_new_user()
   RETURNS trigger AS $$
   BEGIN
     -- メタデータからの値取得とデフォルト値設定
     INSERT INTO public.profiles (
       id,
       username,
       email,
       display_name,
       created_at,
       updated_at
     )
     VALUES (
       NEW.id,
       COALESCE(
         NEW.raw_user_meta_data->>'username',
         'user_' || substr(NEW.id::text, 1, 8)
       ),
       NEW.email,
       COALESCE(
         NEW.raw_user_meta_data->>'display_name',
         'New User'
       ),
       NOW(),
       NOW()
     )
     ON CONFLICT (id) DO NOTHING;  -- 重複防止

     RETURN NEW;
   EXCEPTION
     WHEN OTHERS THEN
       -- エラーログ記録（Supabaseログに出力）
       RAISE LOG 'Error in handle_new_user: %', SQLERRM;
       RETURN NEW;  -- エラーでも認証は継続
   END;
   $$ LANGUAGE plpgsql SECURITY DEFINER;

   -- トリガー作成
   DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users;
   CREATE TRIGGER on_auth_user_created
     AFTER INSERT ON auth.users
     FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
   ```

3. **RLSポリシー設定**

   ```sql
   -- RLS有効化
   ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

   -- 読み取りポリシー：publicプロファイルは誰でも、その他は認証済みのみ
   CREATE POLICY "Public profiles are viewable by everyone"
     ON public.profiles FOR SELECT
     USING (status = 'active');

   -- 更新ポリシー：自分のプロファイルのみ
   CREATE POLICY "Users can update own profile"
     ON public.profiles FOR UPDATE
     USING (auth.uid() = id)
     WITH CHECK (auth.uid() = id);
   ```
