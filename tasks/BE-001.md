本タスクは、RovRov SNSアプリケーションの基盤となるSupabaseプロジェクトの初期設定を行います。これはフェーズ1の最重要タスクであり、後続のすべてのバックエンド開発の前提条件となります。実装は段階的アプローチを採用し、各ステップで検証を行いながら進めることで、堅牢な基盤を構築します。

### 技術的アプローチ

#### 1. プロジェクト構成戦略

- **Infrastructure as Code**: SQL migrationファイルとして設定を保存し、再現性を確保
- **セキュリティファースト**: 最小権限原則に基づいたアクセス制御設定

#### ステップ4: Storageバケット作成（20分）
```typescript
// setup-storage.ts
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_KEY!
);

async function setupStorageBuckets() {
  const buckets = [
    { 
      name: 'avatars', 
      public: true, 
      fileSizeLimit: 5242880, // 5MB
      allowedMimeTypes: ['image/jpeg', 'image/png', 'image/webp']
    },
    { 
      name: 'headers', 
      public: true, 
      fileSizeLimit: 10485760, // 10MB
      allowedMimeTypes: ['image/jpeg', 'image/png', 'image/webp']
    },
    { 
      name: 'posts', 
      public: false, 
      fileSizeLimit: 52428800, // 50MB
      allowedMimeTypes: ['image/*', 'video/*']
    },
    { 
      name: 'temporary', 
      public: false, 
      fileSizeLimit: 10485760, // 10MB
      allowedMimeTypes: ['*']
    }
  ];

  for (const bucket of buckets) {
    const { data, error } = await supabase.storage.createBucket(
      bucket.name, 
      {
        public: bucket.public,
        fileSizeLimit: bucket.fileSizeLimit,
        allowedMimeTypes: bucket.allowedMimeTypes
      }
    );
    
    if (error && !error.message.includes('already exists')) {
      throw error;
    }
  }
}
```

#### ステップ5: バケットポリシー設定（30分）
```sql
-- Storageポリシー設定（SQL Editor）
-- avatarsバケット: 認証ユーザーのみアップロード、全員読み取り可能
CREATE POLICY "Avatar images are publicly accessible"
ON storage.objects FOR SELECT
USING (bucket_id = 'avatars');

CREATE POLICY "Users can upload their own avatar"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'avatars' 
  AND auth.uid()::text = (storage.foldername(name))[1]
);

-- headersバケット: avatarsと同様のポリシー
CREATE POLICY "Header images are publicly accessible"
ON storage.objects FOR SELECT
USING (bucket_id = 'headers');

CREATE POLICY "Users can upload their own header"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'headers' 
  AND auth.uid()::text = (storage.foldername(name))[1]
);

-- postsバケット: 認証ユーザーのみアクセス可能
CREATE POLICY "Post media requires authentication to view"
ON storage.objects FOR SELECT
USING (bucket_id = 'posts' AND auth.role() = 'authenticated');

CREATE POLICY "Users can upload post media"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'posts' 
  AND auth.role() = 'authenticated'
);

-- temporaryバケット: 認証ユーザーのみ、24時間後自動削除
CREATE POLICY "Temporary files for authenticated users"
ON storage.objects FOR ALL
USING (
  bucket_id = 'temporary' 
  AND auth.role() = 'authenticated'
  AND auth.uid()::text = (storage.foldername(name))[1]
);
```

1. **サービスキー漏洩**
   - リスク: GitHubへの誤コミット
   - 緩和策: pre-commitフック、環境変数スキャン、GitHub Secrets使用

2. **不適切なCORS設定**
   - リスク: クロスサイトリクエストフォージェリ
   - 緩和策: 明示的なOrigin許可リスト、本番環境での厳格な制限

   - リスク: ポリシー設定ミスによるデータ漏洩

   - 緩和策: Infrastructure as Code、環境別設定ファイル管理

2. **バックアップ不足**
   - リスク: 設定ミスによるデータ損失
