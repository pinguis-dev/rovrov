### 詳細実装

1. **authStore.ts**:

   ```typescript
   import { create } from 'zustand';
   import { persist, createJSONStorage } from 'zustand/middleware';
   import AsyncStorage from '@react-native-async-storage/async-storage';
   import { supabase } from '../services/supabase';
   import type { User, Session } from '@supabase/supabase-js';

   interface AuthState {
     user: User | null;
     session: Session | null;
     isLoading: boolean;
     isInitialized: boolean;
     signIn: (email: string) => Promise<void>;
     signOut: () => Promise<void>;
     refreshSession: () => Promise<void>;
     initialize: () => Promise<void>;
     setUser: (user: User | null) => void;
     setSession: (session: Session | null) => void;
     setLoading: (loading: boolean) => void;
   }

   export const useAuthStore = create<AuthState>()(
     persist(
       (set, get) => ({
         user: null,
         session: null,
         isLoading: false,
         isInitialized: false,

         signIn: async (email: string) => {
           set({ isLoading: true });
           try {
             const { error } = await supabase.auth.signInWithOtp({
               email,
               options: {
                 emailRedirectTo: 'rovrov://auth/callback',
               },
             });
             if (error) throw error;
           } catch (error) {
             console.error('Sign in error:', error);
             throw error;
           } finally {
             set({ isLoading: false });
           }
         },

         signOut: async () => {
           set({ isLoading: true });
           try {
             const { error } = await supabase.auth.signOut();
             if (error) throw error;
             set({ user: null, session: null });
           } catch (error) {
             console.error('Sign out error:', error);
             throw error;
           } finally {
             set({ isLoading: false });
           }
         },

         refreshSession: async () => {
           try {
             const { data, error } = await supabase.auth.refreshSession();
             if (error) throw error;
             if (data.session) {
               set({ session: data.session, user: data.user });
             }
           } catch (error) {
             console.error('Refresh session error:', error);
             // セッション無効な場合はログアウト処理
             set({ user: null, session: null });
           }
         },

         initialize: async () => {
           try {
             const {
               data: { session },
               error,
             } = await supabase.auth.getSession();
             if (error) throw error;
             set({ session, user: session?.user || null, isInitialized: true });
           } catch (error) {
             console.error('Initialize auth error:', error);
             set({ isInitialized: true });
           }
         },

         setUser: (user: User | null) => set({ user }),
         setSession: (session: Session | null) => set({ session, user: session?.user || null }),
         setLoading: (isLoading: boolean) => set({ isLoading }),
       }),
       {
         name: 'auth-storage',
         storage: createJSONStorage(() => AsyncStorage),
         partialize: (state) => ({
           user: state.user,
           session: state.session,
         }),
       },
     ),
   );
   ```

2. **Auth Listener Setup**:

   ```typescript
   // hooks/useAuthListener.ts
   import { useEffect } from 'react';
   import { supabase } from '../services/supabase';
   import { useAuthStore } from '../store/authStore';

   export const useAuthListener = () => {
     const { setUser, setSession, initialize } = useAuthStore();

     useEffect(() => {
       // 初期化
       initialize();

       // Auth状態変更リスナー
       const {
         data: { subscription },
       } = supabase.auth.onAuthStateChange(async (event, session) => {
         console.log('Auth event:', event);

         switch (event) {
           case 'SIGNED_IN':
           case 'TOKEN_REFRESHED':
             setSession(session);
             break;
           case 'SIGNED_OUT':
             setUser(null);
             setSession(null);
             break;
           case 'USER_UPDATED':
             if (session) {
               setSession(session);
             }
             break;
         }
       });

       return () => {
         subscription.unsubscribe();
       };
     }, [initialize, setUser, setSession]);
   };
   ```

3. **認証ガードHook**:

   ```typescript
   // hooks/useRequireAuth.ts
   import { useEffect } from 'react';
   import { useNavigation } from '@react-navigation/native';
   import { useAuthStore } from '../store/authStore';

   export const useRequireAuth = () => {
     const { user, isInitialized, isLoading } = useAuthStore();
     const navigation = useNavigation();

     useEffect(() => {
       if (isInitialized && !isLoading && !user) {
         navigation.navigate('Auth');
       }
     }, [user, isInitialized, isLoading, navigation]);

     return {
       user,
       isAuthenticated: !!user,
       isLoading: !isInitialized || isLoading,
     };
   };

   // HOC版
   export const withAuthGuard = <P extends object>(
     Component: React.ComponentType<P>
   ): React.FC<P> => {
     return (props: P) => {
       const { user, isLoading } = useRequireAuth();

       if (isLoading) {
         return <LoadingScreen />;
       }

       if (!user) {
         return null; // ナビゲーションが処理される
       }

       return <Component {...props} />;
     };
   };
   ```

### 3. 技術的決定事項

#### 3.1 永続化戦略

**選択:** Zustand persist + AsyncStorage
**理由:**

- React Nativeの標準的なアプローチ
- Zustandとのシームレスな統合
- 非同期処理の最適化
- 設定の柔軟性（partialize機能）

**代替案の検討:**

- SecureStore: 機密性は高いが、パフォーマンスが劣る
- Redux Persist: オーバーヘッドが大きい

#### 3.2 状態管理アーキテクチャ

**設計原則:**

1. **単一責任の原則**: authStoreは認証状態のみを管理
2. **不変性の保持**: 状態更新は常に新しいオブジェクトを生成
3. **予測可能性**: アクションの結果が予測可能
4. **テスタビリティ**: モックやスタブが容易

#### 3.3 セキュリティ考慮事項

**データ保護:**

- セッション情報の安全な保存
- トークンの適切なライフサイクル管理
- エラー情報の適切な処理（機密情報の非露出）

**アクセス制御:**

- 認証状態の検証プロセス
- 未認証アクセスの適切なブロック
- セッション期限切れの自動検出

## 実装方針（Implementation Strategy）

### 概要

本実装は、既存のauthStore実装をZustand persistミドルウェアベースに再構築し、Supabase Auth Listenerとの統合、認証ガード機能の追加を行います。現在の実装状況を踏まえ、段階的かつ安全な移行アプローチを採用します。

### 現状分析

#### 既存実装の評価

- **authStore.ts**: 基本機能実装済み、ただしpersistミドルウェア未実装
- **secureStorage**: authStorageとして実装済み（手動永続化）
- **App.tsx**: checkSession実装済み、ただしAuth Listener未実装
- **ナビゲーション**: 未実装（index.tsのみ存在）
- **型定義**: User, Session, AuthError完備

#### ギャップ分析

1. **永続化**: 手動実装 → Zustand persistミドルウェアへの移行必要
2. **Auth Listener**: 未実装 → リアルタイム認証状態同期必要
3. **認証ガード**: 未実装 → useRequireAuth Hook必要
4. **初期化フロー**: 部分実装 → 完全な初期化シーケンス必要

### 実装フェーズ
